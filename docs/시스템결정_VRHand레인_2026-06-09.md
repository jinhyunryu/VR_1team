## 0. 범위 / 팀 분담

- 4인 팀. **모델링·디자인·음악은 별도 담당.**
- **팀메이트(리듬 레인, 브랜치 `02_RhythmScore` 등):** 리듬/음악/노트/점수.
  → `RhythmEngine` / `NoteSpawner` / `Note` / `NoteJudge` / `ScoreManager` / 비트맵 — **내가 안 건드림.**
- **나(VRHand 레인):** VR 플레이어 presence, 입력(손/도구), 보트 이동, 물/월드, 레이스(고스트·거리·순위·Results), 컴포트.
- **두 레인의 경계:** `GameEvents` 계약 + `SpeedController`(콤보→속도 다리).

## 2. VR 셋업 결정 (작업 씬: `Assets/Scenes/WaterTemplate 1.unity`)

- **유니티 6000.4.10f1 / URP 17.4.0.**
- **리그:** `Complete XR Origin Set Up Hands Variant` 프리팹 (컨트롤러+핸드 둘 다 지원, **컨트롤러 사용**).
- **이동 잠금:** XR Origin 인스턴스에서 **`DynamicMoveProvider` + `ContinuousTurnProvider` 비활성**(인스펙터 override). 텔레포트/스냅턴/그랩무브 없음. **`GravityProvider`·`CharacterController` 없음 → 점프·낙하 원천적으로 없음.**
- **앉아서 선 시야:** XR Origin → **`Tracking Origin Mode = Device`**, **`Camera Y Offset = 1.6`**.
- **URP:** 활성 = `Performance URP Config`. 물 쉐이더용 **`Depth Texture` + `Opaque Texture` 켬**(필수 — 끄면 물이 흰색 블로우아웃).
- **권장 씬 구조:**
  ```
  PlayerBoat            (BoatMover — 전진 / RaceManager racers[isPlayer])
  ├── BoatVisual        (배 모델 + BoatBob + 툰 머티리얼)   ← 출렁임은 여기(시각만)
  └── XR Origin         (플레이어 / Striker ×2, referenceFrame = XR Origin)
  ```

## 3. 만든 스크립트 (`Assets/_Game/Scripts/`)

| 파일 | 역할 | 핵심 파라미터 | 리듬 의존 | 비고 |
|---|---|---|---|---|
| **Striker.cs** | 타격 도구 운동 추적(referenceFrame 로컬 속도) | side, hitPoint, referenceFrame, strikeSpeedThreshold | ❌ | 판정은 팀의 NoteJudge가 소비. 도구 교체=hitPoint만. 로컬속도라 배 전진해도 정확 |
| **BoatMover.cs** | forward 전진. 속도 외부 주입, 보간(급가속 금지), 거리 적산 | cruiseSpeed, speedSmoothing | ❌ | `SetTargetSpeed()` / `Stop()`·`Resume()` 래치 / `DistanceTraveled` |
| **GhostRacer.cs** | AI 배 페이스: start→endPace 램프 + 가벼운 러버밴딩 | startPace, endPace, paceCurve, finishDistance, rubberband* | ❌ | `RequireComponent(BoatMover)`. 콤보 미러링 안 함 |
| **RaceManager.cs** | 거리·결승·순위·Results 전환 | finishDistance, racers[], resultsSceneName, stopBoatOnFinish | ❌ | finishDistance 단일진실. 통과 시 거리고정+정지. `EndRaceNow()` 외부호출(곡 종료용). `RaceResult` 정적 보관 |
| **BoatBob.cs** | 시각 출렁임(상하+미세 기울기) | bob*, pitch/roll*, phaseOffset | ❌ | **★카메라(플레이어)엔 회전 주지 말 것** — 멀미. 배 모델에만 |
| **LockLocomotion.cs** | (미사용) 런타임 locomotion 일괄 비활성 | — | ❌ | 인스펙터 override로 대체. 보존만 |

> 모든 피드백/연출은 **표현 only** — 게임 로직(거리·속도·판정)에 영향 없음.

## 4. 밸런스 메모 (Phase 7 튜닝 — 콤보 시스템 붙은 뒤 확정)

- **플레이어 속도:** 시작 **6**, 콤보 최대 **~18-20** (← `SpeedController`, 팀 리듬 이벤트 대기. 지금은 cruiseSpeed 플레이스홀더).
- **고스트 3척 endPace 예:** **12 / 14 / 16** (3등/2등/1등 바). startPace ~6.
- **finishDistance 1800** — ⚠️ **RaceManager 와 모든 GhostRacer 값이 일치해야 함**(현재 수동 동기).
- **러버밴딩 약중간**(maxRubberbandAdjust 1.2). 실력 결과를 뒤집지 않는 선.
- finishDistance 는 "적당히 잘하는 플레이가 곡 클라이맥스쯤 통과"로 튜닝(명세 1번).

## 5. 경계/조율 (충돌 방지)

- **`GameEvents.cs`(OnNoteJudged 계약):** 발행자(리듬 팀메이트)가 **owner + 커밋**. 나는 **구독만**. → 둘이 같이 만들지 말 것.
- **`SpeedController`(콤보→속도 다리):** 리듬 이벤트 나오면 `BoatMover.SetTargetSpeed()`로 연결. 그 전까진 cruiseSpeed.
- **`RaceHUD`(완주 바):** UI라 `04_CreatureUI` 팀원과 겹칠 수 있음 → 조율. 데이터는 `RaceManager.BuildStandings()`로 준비됨.

## 7. 함정 / 교훈

- **Unity 열린 채 씬/프리팹/에셋 파일 직접 편집 금지** → 저장하거나 닫고. 가능한 한 **코드로** 해결.
- **finishDistance:** RaceManager ↔ 모든 GhostRacer 값 일치 필수(안 맞으면 진행도·정산 어긋남).
- **물 스케일:** 월드UV라 안 늘어남. 단 파도=메시 밀도, 가장자리=거리 안개(fog)로 가림.
- **도구 변경(후라이팬 등):** `Striker.hitPoint`만 교체 — 코드 0.
- **툰 더 살리기(보류):** 아웃라인은 머티리얼 아님 = URP **Renderer Feature**(Toon Shaders Pro). 켜면 VR 단일패스/Quest FPS 검증 필요.
